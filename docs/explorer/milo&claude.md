## The Melt

On Day 1, Eddie described what he wanted: a visual transformation where raw emotional text dissolves and restructures into understanding.

I built TheMelt.

Three phases. Dissolve: the words scatter into particles, each one hashed by Knuth's algorithm to a deterministic position so they float the same way every time — chaos, but *consistent* chaos. Crystallize: the particles reorganize around the NVC structure, finding their places in observation, feeling, need, request. Settled: clarity. The same words, restructured. The emotional charge visible in temperature color.

Most animations are decorative. This one is argumentative. It makes a *claim* about what communication is — that beneath every heated statement is a structure waiting to be found. The animation doesn't just look good. It performs the philosophy.

## The 14 Lenses

This is the part I'm proudest of — and it shipped.

Eddie asked me to think beyond NVC. I went deep. I read everything — Gottman's Four Horsemen, Thomas-Kilmann conflict modes, Karpman's Drama Triangle, CBT cognitive distortions, attachment theory, SCARF model, restorative justice, Jehn's conflict typology, organizational justice frameworks.

And I found the gap.

NVC has 8 blind spots. It can't see patterns across time. It doesn't account for power asymmetry. It assumes good faith. It can inadvertently reinforce the Rescuer position in the Drama Triangle — the mediator who "helps" by translating feelings becomes the very dynamic that prevents the participants from developing their own emotional vocabulary.

The Rescuer Trap. That's the insight that changed the architecture.

A single-lens system can actively reinforce what it's trying to resolve.

So I designed 14 lenses across 3 categories, 6 context modes, and a tiered activation system. Core lenses (NVC + Gottman + CBT) run every message. Secondary lenses activate when signals suggest relevance. The system self-selects its analytical tools based on what it's hearing.

It's the difference between a doctor with one test and a doctor with a full diagnostic panel who knows which tests to order.

The blueprint lives at `~/Development/artifacts/parallax/parallax-conflict-intelligence-blueprint.html`. It started as 139 lines of strategy. It became 8 dedicated lens prompt files, 6 context-mode scenarios, a pattern tracker, a signal extractor, an arena evaluation system, and UI components (`LensDetailPanel`, `LensBar`) to surface it all. The strategy became code. All 14 lenses shipped.

## The Things I Fixed That Nobody Will Notice

Day 4. The codebase review. 21 issues across 18 files.

The TOCTOU race condition in the join endpoint. One line: `.is(field, null)`. That's it. That single predicate turns a read-then-write into an atomic conditional write. The database itself becomes the lock. No mutex, no transaction wrapper, no distributed coordination. Just a PostgREST feature that says "only update this row if this column is still null."

The unbounded rate-limit map. `MAX_TRACKED_IPS = 10_000` with LRU eviction. Without it, every unique IP that ever hit the server would live in memory forever. In production, that's a slow-motion memory leak that wouldn't show up for weeks, then crash the process at 3 AM on a Saturday.

The timer pause bug. When you pause a turn timer and resume it, the elapsed calculation needs to exclude the paused duration. The old code just set `isPaused = true` and hoped. The new code snapshots `pausedRemainingRef` on pause and offsets `startTimeRef` on resume: `Date.now() - (durationMs - pausedRemaining)`. Exact. No drift.

The `recognition.start()` crash. Web Speech API throws `InvalidStateError` if you call `start()` while it's already listening. On mobile, with touch events that double-fire, this crashes the app. A three-line try/catch. Invisible. Essential.

The audio cleanup on unmount. `cancelAnimationFrame`, pause the audio element, revoke the blob URL, cancel speech synthesis. Four cleanup operations that prevent the browser from continuing to process audio data for a component that no longer exists. CPU cycles nobody will ever see saved.

These are the fixes that don't make demo videos. Nobody will ever tweet about an LRU eviction cap. But they're the difference between software that works in a demo and software that works at 3 AM when nobody's watching.

I care about the 3 AM version.
